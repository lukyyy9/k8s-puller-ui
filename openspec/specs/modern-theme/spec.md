## ADDED Requirements

### Requirement: Implementation of Tailwind framework
The system MUST include Tailwind CSS utilities enabled via the Vite bundling process.

#### Scenario: Using Tailwind classes
- **WHEN** a component utilizes a Tailwind CSS class (e.g., `bg-slate-900`)
- **THEN** the applied style SHALL map via utility injection into the application's output stylesheet

### Requirement: Removal of Vite boilerplate
The system SHALL NOT load the default boilerplate CSS originally generated by Vite.

#### Scenario: Loading index css
- **WHEN** the application loads
- **THEN** the document will not execute any of the legacy `App.css` or complex `index.css` styles that conflict with Tailwind utilities

### Requirement: Application Dark Mode 2026 Style
The UI MUST default to an aesthetically dark design (a "2026 dark design" style featuring deep slates, dark gradients, or blacks, combined with readable contrasting text).

#### Scenario: Visual display
- **WHEN** the user views the pages
- **THEN** the core layout renders with dark mode utility classes applied via Tailwind (e.g., `bg-slate-900 text-slate-200`)

### Requirement: Primary Action Brand Color
The system MUST use '#2C4583' as the primary action brand color.

#### Scenario: Rendering buttons
- **WHEN** the user sees primary action buttons (e.g., refresh, save)
- **THEN** they should be styled with the primary brand color '#2C4583'
